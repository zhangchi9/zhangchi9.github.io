Matlab, “parpool not enough input argument” solution

Recently when I used matlab parallel computing on my university’s cluster with matlab 2016a on Unix OS, I was trying to open a parpool using parpool(4). Unexpectedly it gives me an error: ”not enough input argument”. The solution is given by the links. Thanks to randy cruise for providing this solution. To summarize:
  • In matlab command window type in “prefdir”
  • You can see a directory looks like C:\…….\MATLAB\R2016a
  • Cd to C:\…….\MATLAB
  • Delete the folder “local_cluster_jobs” and restart matlab.
